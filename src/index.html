Методология оценки4 балла

Эволюция метрик качества

Переход от синтетических тестов к оценке решения реальных инженерных задач.

1. SWE-bench Verified: Новый отраслевой стандарт (взамен HumanEval). Оценивает способность агента автоматически решать Issues из реальных репозиториев GitHub (воcпроизведение бага → создание патча → прохождение тестов).

2. CVE-Bench (Безопасность): Специализированный бенчмарк 2025 года для оценки способности моделей исправлять уязвимости (CVE) без внесения новых дефектов.

Архитектура Агентов8 баллов

Паттерны проектирования агентов

Отказ от простой генерации (Zero-shot) в пользу итеративных алгоритмов.

Рефлексия (Reflection)

Агент анализирует логи компиляции или вывода линтера, самостоятельно находит свои ошибки и исправляет код в цикле до успешного прохождения тестов.

Использование инструментов (Tool Use)

Способность модели вызывать внешние утилиты (git, grep, docker) для анализа контекста проекта, а не только генерации текста.

Информационная безопасность16 баллов

Риски внедрения и векторы атак

Галлюцинации пакетов (Package Hallucination): Модели могут предлагать импорт несуществующих библиотек, что открывает вектор для атак на цепочки поставок (Supply Chain Attacks).

Решение (Data Governance): Использование локальных шлюзов (AI Gateway) и квантованных моделей (GGUF) внутри закрытого контура предприятия для предотвращения утечки интеллектуальной собственности.
